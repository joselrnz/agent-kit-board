# Archive Policy

AgentKitBoard keeps the README useful first and complete second.

```text
README
├─ Trending
│  ├─ Daily: up to 20
│  ├─ Weekly: up to 20
│  └─ Monthly: up to 20
│
├─ Permanent grouped repo lists
│  └─ Target cap: 100 visible repo rows total
│
└─ docs/archive/
   └─ Useful overflow, older references, and lower-priority repos
```

Current policy:

- Do not cap trending as part of the 100-row permanent-list budget.
- Keep permanent grouped repos at roughly 100 visible rows total.
- Sort permanent groups by practical relevance first, then popularity, freshness, and maintenance signal.
- When permanent groups exceed 100 rows, move weaker or lower-priority references to `docs/archive/`.
- Do not delete useful references just because they leave the README.
- Keep machine-readable trending history in `data/trending-archive.json`.
